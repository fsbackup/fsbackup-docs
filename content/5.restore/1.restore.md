---
title: Restoring data
description: How to restore files from local snapshots or S3 using fs-restore.sh or manually.
---


fsbackup supports two restore paths: from local snapshots and from S3 archives.

## Restore from local snapshot

Use `fs-restore.sh` to restore files from a local snapshot to a destination directory.

### Basic restore

```bash
docker exec -it fsbackup /opt/fsbackup/utils/fs-restore.sh \
  --class class1 \
  --target myapp.data \
  --tier daily \
  --date 2025-03-14 \
  --dest /restore/myapp
```

This copies the snapshot contents to `/restore/myapp` on the backup server.

### Restore to a remote host

```bash
docker exec -it fsbackup /opt/fsbackup/utils/fs-restore.sh \
  --class class1 \
  --target myapp.data \
  --tier daily \
  --date 2025-03-14 \
  --to-host myhost \
  --dest /var/lib/myapp
```

### List available snapshots

```bash
# List daily snapshots for a target
ls /backup/snapshots/daily/*/class1/myapp.data/

# List weekly
ls /backup/snapshots/weekly/*/class1/myapp.data/
```

### Manual restore (without the script)

You can also copy files directly from the snapshot directory:

```bash
# Find the snapshot
ls /backup/snapshots/daily/2025-03-14/class1/myapp.data/

# Copy specific files
cp -a /backup/snapshots/daily/2025-03-14/class1/myapp.data/path/to/file /restore/
```

## Restore from S3

S3 archives are encrypted with age and compressed with zstd. You need the age private key to decrypt.

### Download from S3

```bash
aws s3 cp \
  s3://your-bucket/monthly/class1/myapp.data/myapp.data--2025-03.tar.zst.age \
  myapp.data--2025-03.tar.zst.age \
  --profile fsbackup
```

### Decrypt and extract

```bash
age -d -i /path/to/age.key myapp.data--2025-03.tar.zst.age \
  | zstd -d \
  | tar -xf - -C /restore/myapp/
```

::u-callout{icon="i-lucide-key" color="orange"}
The age private key is **never stored on the backup server**. Store it securely offline. Without it, S3 archives cannot be decrypted.
::

## Finding the right snapshot

| Need | Look in |
|------|---------|
| Latest daily | `daily/<yesterday>/` |
| Last week's state | `weekly/<YYYY-Www>/` |
| Last month's state | `monthly/<YYYY-MM>/` |
| End of last year | `annual/<YYYY>/` |
| Specific deleted file | Browse `daily/` snapshots chronologically |

## Retention — how far back can you go?

| Tier | How far back |
|------|-------------|
| Daily | 14 days |
| Weekly | 8 weeks |
| Monthly | 12 months |
| Annual | Indefinite (S3) |
