---
title: Restoring data
description: How to restore files from local ZFS snapshots or S3 archives using fs-restore.sh or manually.
---


fsbackup supports two restore paths: from local ZFS snapshots and from S3 archives.

## Restore from local snapshot

### List available snapshots

```bash
# All snapshots
sudo -u fsbackup /opt/fsbackup/utils/fs-restore.sh list

# All snapshots for a class
sudo -u fsbackup /opt/fsbackup/utils/fs-restore.sh list --class class1

# Snapshots for a specific target
sudo -u fsbackup /opt/fsbackup/utils/fs-restore.sh list --class class1 --id myapp.data
```

### Restore the latest snapshot

```bash
sudo -u fsbackup /opt/fsbackup/utils/fs-restore.sh restore \
  --class class1 \
  --id myapp.data \
  --latest \
  --dest /restore/myapp
```

### Restore a specific snapshot

```bash
sudo -u fsbackup /opt/fsbackup/utils/fs-restore.sh restore \
  --class class1 \
  --id myapp.data \
  --snapshot daily-2026-03-14 \
  --dest /restore/myapp
```

### Restore the latest snapshot of a specific type

```bash
sudo -u fsbackup /opt/fsbackup/utils/fs-restore.sh restore \
  --class class1 \
  --id myapp.data \
  --latest --type weekly \
  --dest /restore/myapp
```

### Manual restore (without the script)

ZFS snapshots are accessible as read-only directories via `.zfs/snapshot/`:

```bash
# Browse available snapshots
ls /backup/snapshots/class1/myapp.data/.zfs/snapshot/

# Copy specific files
cp -a /backup/snapshots/class1/myapp.data/.zfs/snapshot/daily-2026-03-14/path/to/file /restore/
```

## Restore from S3

S3 archives are encrypted with age and compressed with zstd. You need the age private key to decrypt.

### Find the archive

S3 key format: `<tier>/<class>/<target>/<target>--<date>.tar.zst.age`

```bash
aws s3 ls s3://your-bucket/monthly/class1/myapp.data/ --profile fsbackup
```

### Download from S3

```bash
aws s3 cp \
  s3://your-bucket/monthly/class1/myapp.data/myapp.data--2026-03.tar.zst.age \
  myapp.data--2026-03.tar.zst.age \
  --profile fsbackup
```

### Decrypt and extract

```bash
age -d -i /path/to/age.key myapp.data--2026-03.tar.zst.age \
  | zstd -d \
  | tar -xf - -C /restore/myapp/
```

::u-callout{icon="i-lucide-key" color="orange"}
The age private key is **never stored on the backup server**. Store it securely offline (password manager, encrypted USB). Without it, S3 archives cannot be decrypted.
::

## Finding the right snapshot

| Need | Snapshot type | How far back |
|------|---------------|-------------|
| Most recent files | Daily | Up to `KEEP_DAILY` days (default: 14) |
| Last week's state | Weekly | Up to `KEEP_WEEKLY` weeks (default: 8) |
| Last month's state | Monthly | Up to `KEEP_MONTHLY` months (default: 12) |
| End of a prior year | Annual | All kept (default: indefinite) |
| Specific deleted file | Browse daily snapshots chronologically | — |

Annual snapshots are also exported to S3 for off-site recovery.
